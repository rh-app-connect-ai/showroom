== Lab 4 - AI Customer Support

Use an AI agent to orchestrate backend actions based on natural language chat — powered by Agentic Apache Camel and Model-as-a-Service (MaaS).

*The Business Need*

When a customer reaches out about a purchase order — whether to update details, return items, or resolve a discrepancy — support agents traditionally have to manually navigate multiple backend databases and file systems to address the request. This is slow, error-prone, and frustrating for both agents and customers. They need a faster, frictionless way to handle these cases.

{empty} +

*The Technical Solution*

Support agents interact with an AI bot directly within their Rocket.Chat window using natural language. Powered by Agentic Apache Camel and Model-as-a-Service (MaaS), the AI acts as a smart assistant — it interprets the agent's request, autonomously decides which tools to call, updates the database, and regenerates the corrected invoice in S3 storage. This dramatically reduces resolution time and eliminates manual "swivel-chair" tasks for support teams.

{empty} +

The AI agent follows a reasoning loop — it consults the LLM via MaaS, which decides what actions to take. Camel then executes the appropriate tools (update the database, regenerate a PDF invoice in S3), feeds the results back to the LLM, and returns the final answer to the support agent in Rocket.Chat.

{empty} +

*What you will do in this lab:*

. *Set up the environment* — Prepare the working folder and configure the AI agent.
. *Develop the Agent* — Progressively enable the AI agent, from basic routes to fully-equipped with tools.
. *Complete the flow* — Finish the integration that links Rocket.Chat, the AI agent, the database, and S3 storage.
. *Test the flow* — Simulate a customer/agent interaction and validate the results.
