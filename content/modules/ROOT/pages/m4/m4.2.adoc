{banner-under-construction}

Here's the Camel route you'll be working with. It listens for messages from Rocket.Chat, extracts the text, and sends a response back:

{blank}

----

The `set system prompt` step is disabled — there's no AI agent yet, so no prompt is needed. In its place, the `dummy response` step simply returns "Hello World" to Rocket.Chat. Any message sent to the bot triggers this flow and gets back the placeholder reply.

Your task will be to replace the `dummy response` with a MaaS-powered AI agent, and enable the system prompt that guides its behaviour.

=== Run the starting point

Start by running the Camel route as-is to see the placeholder behaviour in action:

{blank}

[.copypaste]
****
camel forage run \
  agent/* \
  application.properties \
  --dep=mvn:io.kaoto.forage:forage-memory-message-window:1.1-SNAPSHOT \
  --dev
****

[TIP]
--
.What does this command do?
[%collapsible]
====
* `camel forage run` — runs the Camel route using the Forage agent runtime
* `agent/*` — loads all files from the `agent/` folder (route, tools, templates)
* `application.properties` — loads the model configuration and credentials
* `--dep=mvn:io.kaoto.forage:forage-memory-message-window:1.1-SNAPSHOT` — adds the message-window memory dependency at runtime
* `--dev` — dev mode, automatically reloads when you save changes to any source file
====
--

=== Wire the AI agent

Now replace the dummy response with a real AI agent. Open `agent.camel.yaml` in _Kaoto_ and follow the steps below:

{empty} +

. *Enable the system prompt*
+
The `set system prompt` step is currently disabled — it's greyed out because there was no AI agent to use it. Now that you're about to add one, enable it: hover the step and click the {btn-enable} button (next to the {btn-replace} button).
+
{empty} +

. *Replace the dummy response with an Agent*
+
The `dummy response` step is a placeholder — it always returns "Hello World". Replace it with the _LangChain4j Agent_ component that connects to MaaS.
+

1. Hover the `dummy response` step and click the floating {btn-replace} button.
1. Find and select the _LangChain4j Agent_ component.
1. Select the `All` configuration tab.
1. Set the `Agent Id` to:
+
[.copypaste]
****
agent
****
+
Save the file. Because you're running in dev mode, Camel hot-reloads automatically. Watch the terminal — the logs should confirm the agent is running:

[TIP]
--
.Why did the bot respond that way?
[%collapsible]
====
The `system.message` file contains the system prompt that guides the AI agent's behaviour. One of its rules explicitly instructs the model to decline when it lacks the right tool:

----
# CRITICAL CONSTRAINTS
* **NO TOOL, NO URL:** If `get_invoice_pdf` is missing from your
  current toolset, respond EXACTLY: "I'm sorry, I don't have the
  capability to retrieve that PDF right now."
----

The model follows this constraint because it can see which tools are available. Right now it has none — so it politely refuses. Once you register the tools, the model will use them instead.
====
--

=== Hook the tools

The route already includes a predefined tool (`get_invoice_pdf`) — but the agent doesn't know about it yet. Camel uses _tags_ to match agents with their tools: when an agent's tag matches a tool's tag, the tool becomes available.

. *Configure the agent's tags*
+

1. Click the `LangChain4j Agent` step to open its configuration.
1. Select the `All` configuration tab.
1. Set the `Tags` property to:
+
[.copypaste]
****
all
****
+
Save the file. Camel hot-reloads and the agent now has access to the tool. Try it — go back to _Rocket.Chat_ and ask:

{blank}

[.copypaste]
****
What about now?
****

This time the LLM discovers the tool, calls it, and responds with the invoice:

=== Amend the invoice

The agent can do more than just retrieve data — it can act on it. Imagine the customer has changed their mind and wants to return one of the items. Ask the AI assistant to amend the invoice:

{blank}

[.copypaste]
****
The customer has changed his mind and wants to return one of the items purchased. Could you please amend the invoice?
****

The agent inspects the invoice, identifies the matching item, triggers the necessary tools to update the database, and responds with a summary of the changes:

{blank}

:fig-time: 6:49 PM
:fig-user-msg: The customer has changed his mind and wants to return one of the items purchased. Could you please amend the invoice?

The agent acted autonomously — it inspected the invoice, identified the item, updated the quantity in the database, recalculated the totals, and confirmed the changes. All from a single natural language request.

=== Raise a ticket

A customer support process rarely ends with a single action. Returning an item involves multiple departments — confirming the goods were received, processing the refund, updating inventory, notifying the customer. Rather than handling each step manually, ask the AI agent to raise a ticket that coordinates everything:

{blank}

[.copypaste]
****
Could you please raise a ticket? Include the details explained, and ask the right departments to refund the customer accordingly.
****

The agent gathers the context from the conversation, creates a ticket in the HelpDesk system, and responds with a confirmation:

{blank}

:fig-time: 6:52 PM
:fig-user-msg: Could you please raise a ticket? Include the details explained, and ask the right departments to refund the customer accordingly.
:fig-bot-msg: A ticket has been created with ID 1025. Would you like to share this ticket with the customer?
include::_figures/rc-interaction.adoc[]

At this point, the bot may behave in different ways — LLMs are unpredictable. Here's what to expect and how to respond:

* **The bot asks to share the ticket** (as shown above) — reply with:
+
[.copypaste]
****
Yes, please share it with the customer.
****

* **The bot shares the ticket automatically** without asking — no action needed, skip ahead.

* **The bot doesn't mention sharing at all** — ask it explicitly:
+
[.copypaste]
****
Could you share the ticket with the customer?
****

Either way, the final outcome is the same — the bot shares the ticket and the customer receives a link to the HelpDesk system:
