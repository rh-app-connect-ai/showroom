=== Start the dev loop

From this point on, you'll work in a rapid development cycle — _Camel JBang_ runs in dev mode, which means every change you make to the route is instantly picked up and applied. You can modify the flow, save the file, and immediately observe the new behaviour without restarting.

Start by running the Camel route as-is to see the placeholder behaviour in action:

{blank}

[.copypaste]
****
camel run * \
--dev \
--local-kamelet-dir ../support/kamelets
****

[TIP]
--
.What does this command do?
[%collapsible]
====
* `camel run *` — runs all Camel route files in the current directory
* `--dev` — dev mode, automatically reloads when you save changes to any source file
* `--local-kamelet-dir ../support/kamelets` — loads custom Kamelets from the support folder
====
--

[TIP]
--
.Why did the bot respond that way?
[%collapsible]
====
In short, the tools exist in the project but the AI agent can't see them yet.

The `system.message` file contains the system prompt that guides the AI agent's behaviour. One of its rules explicitly instructs the model to decline when it lacks the right tool:

----
# CRITICAL CONSTRAINTS
* **NO TOOL, NO URL:** If there is no tool fit for the job (missing),
  you MUST return Pattern 2 with the text: "I'm sorry, I don't have 
  the capability to retrieve that PDF right now."
----

Once you register the tools, the agent will discover and use them automatically.
====

=== Hook the tools

The project includes a number of tools that integrate with the necessary backends — but the agent doesn't know about them yet. Camel uses _tags_ to match agents with their tools: when an agent's tag matches a tool's tag, the tool becomes available.

[TIP]
--
.How are tools defined in Apache Camel?
[%collapsible]
====
In Camel, a tool is just a regular route that starts with the `langchain4j-tools` component. The route describes what the tool does — the agent discovers it automatically via tags and invokes it when needed.

For example, here's one of the project's tool routes — it retrieves the PDF URL for a given invoice:

image::m4/kaoto-tool-route.png[width=50%]

The route starts with the `langchain4j-tools` source, processes the request through a series of steps, and returns the result to the LLM. No special wiring is needed — once the tags match, the agent can call the tool.
====
--

=== Raise a ticket

A customer support process rarely ends with a single action. Returning an item involves multiple departments — confirming the goods were received, processing the refund, updating inventory, notifying the customer. Rather than handling each step manually, ask the AI agent to raise a ticket that coordinates everything:

{blank}

[.copypaste]
****
Could you please raise a ticket? Include the details explained, and ask the right departments to refund the customer accordingly.
****

The agentic system gathers the context from the conversation, creates a ticket in the HelpDesk system, and responds with a confirmation:

{blank}

:fig-time: 6:52 PM
:fig-user-msg: Could you please raise a ticket? Include the details explained, and ask the right departments to refund the customer accordingly.
:fig-bot-msg: A ticket has been created with ID 1025. Would you like to share this ticket with the customer?
include::_figures/rc-interaction.adoc[]

Remember, _Matrix_ represents the customer side, while _Rocket.Chat_ is where the support agent works. If you haven't done this yet, follow the xref:m2/m2.1.adoc[Matrix platform onboarding] instructions.
