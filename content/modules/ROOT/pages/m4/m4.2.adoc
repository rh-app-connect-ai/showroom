== Building the Agentic Flow

Here's the Camel route you'll be working with. It listens for messages from Rocket.Chat, extracts the text, and sends a response back:

{blank}

[.mermaid]
----
flowchart LR
    A(RocketChat) --> B(get text) --> C(set system prompt) --> D2 --> E(to rocketchat)

    subgraph D2 ["To do: Agent"]
        D(dummy response)
    end

    style C fill:#eee,stroke:#ccc,color:#aaa
    style D2 fill:#dde4f6,stroke:#6690e6,color:#335
----

The `set system prompt` step is disabled — there's no AI agent yet, so no prompt is needed. In its place, the `dummy response` step simply returns "Hello World" to Rocket.Chat. Any message sent to the bot triggers this flow and gets back the placeholder reply.

Your task will be to replace the `dummy response` with a MaaS-powered AI agent, and enable the system prompt that guides its behaviour.

{empty} +

=== Run the starting point

Start by running the Camel route as-is to see the placeholder behaviour in action:

{blank}

[.copypaste]
****
camel forage run \
  agent/* \
  application.properties \
  --dep=mvn:io.kaoto.forage:forage-memory-message-window:1.1-SNAPSHOT \
  --dev
****

[TIP]
--
.What does this command do?
[%collapsible]
====
* `camel forage run` — runs the Camel route using the Forage agent runtime
* `agent/*` — loads all files from the `agent/` folder (route, tools, templates)
* `application.properties` — loads the model configuration and credentials
* `--dep=mvn:io.kaoto.forage:forage-memory-message-window:1.1-SNAPSHOT` — adds the message-window memory dependency at runtime
* `--dev` — dev mode, automatically reloads when you save changes to any source file
====
--

=== Wire the AI agent

Now replace the dummy response with a real AI agent. Open `agent.camel.yaml` in _Kaoto_ and follow the steps below:

{empty} +

. *Enable the system prompt*
+
The `set system prompt` step is currently disabled — it's greyed out because there was no AI agent to use it. Now that you're about to add one, enable it: hover the step and click the {btn-enable} button (next to the {btn-replace} button).
+
{empty} +

. *Replace the dummy response with an Agent*
+
The `dummy response` step is a placeholder — it always returns "Hello World". Replace it with the _LangChain4j Agent_ component that connects to MaaS.
+

1. Hover the `dummy response` step and click the floating {btn-replace} button.
1. Find and select the _LangChain4j Agent_ component.
1. Select the `All` configuration tab.
1. Set the `Agent Id` to:
+
[.copypaste]
****
agent
****
+

[TIP]
--
.Why did the bot respond that way?
[%collapsible]
====
The `system.message` file contains the system prompt that guides the AI agent's behaviour. One of its rules explicitly instructs the model to decline when it lacks the right tool:

----
# CRITICAL CONSTRAINTS
* **NO TOOL, NO URL:** If `get_invoice_pdf` is missing from your
  current toolset, respond EXACTLY: "I'm sorry, I don't have the
  capability to retrieve that PDF right now."
----

The model follows this constraint because it can see which tools are available. Right now it has none — so it politely refuses. Once you register the tools, the model will use them instead.
====
--

+

1. Click the `LangChain4j Agent` step to open its configuration.
1. Select the `All` configuration tab.
1. Set the `Tags` property to:
+
[.copypaste]
****
all
****
+
