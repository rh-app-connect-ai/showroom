== Diagram Styling Notes

=== Color Palette (Red Hat themed)

* **Primary red**: `#EE0000` — used for icon strokes, accents, subgraph labels, highlights
* **Light red background**: `#fce4e4` — used for inner boxes (e.g. Integration steps)
* **Darker red accents**: `#cc0000` / `#ff4444` — used for shading on solid fills (e.g. database cylinder)
* **Light grey background**: `#f5f5f5` — used for icon fills (invoice, record, S3)
* **Grey details**: `#bbb` / `#ccc` / `#e0e0e0` — used for secondary lines and shading

=== SVG Icon Conventions

* All icons stored in `assets/images/test/` (self-contained for drafting)
* All icon strokes use `#EE0000` for consistency
* Icons referenced via `@{ img: "../_images/test/<name>.svg" }` in Mermaid
* Node boxes hidden with `style <id> fill:none,stroke:none`
* Labels placed below icons with `pos: "b"`
* Standard size: `h: 80, w: 80` (adjust per icon aspect ratio)

=== Subgraph Tricks

* **Label outside, box inside**: make the outer subgraph invisible (`fill:none,stroke:none`) and style the inner node as the visible box
* **Bold label clipping**: Mermaid miscalculates bold text width — add a trailing `&amp;nbsp;` to the label (e.g. `["OpenShift &amp;nbsp;"]`). More reliable than trailing spaces which may be trimmed
* **Numbered edge labels**: use circled numbers (`①②③`) to link diagram paths to a numbered list inside a subgraph
