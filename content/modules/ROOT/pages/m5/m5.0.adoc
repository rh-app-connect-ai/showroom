== Lab 5 - Protect LLM consumption with Connectivity Link

== Introduction

In the previous phases of this workshop, you connected disparate chat platforms into a unified communication hub, securely ingested and transformed client data, and introduced an AI agent that helps support agents resolve invoice issues through natural language -- all powered by _Apache Camel_, _Kafka_, and Model-as-a-Service (MaaS).

But providing internal teams with access to Large Language Models introduces new challenges. AI resource costs are tied to **token consumption** rather than simple API hits. Without proper governance, there is no way to prevent system abuse or control runaway expenses. Traditional rate limiting -- which simply counts requests -- is not enough.

In this final phase, you will place a **governance layer** in front of the LLM using _Red Hat Connectivity Link_. You will:

* **AuthPolicy** -- Control _who_ can access the LLM API using API key-based authentication and group-based authorization.
* **TokenRateLimitPolicy** -- Control _how much_ each user can consume by enforcing rate limits based on actual LLM token usage.

By the end of this module, the AI agent will be fully secured with tiered access control -- _free_ users get a limited token budget, while _gold_ users enjoy a higher allowance.

== Introducing Red Hat Connectivity Link

=== What is Connectivity Link?

_Red Hat Connectivity Link_ is a _Kubernetes_-native solution for managing, securing, and controlling API traffic. Built on the _Kuadrant_ project, it integrates directly with the _Gateway API_ to enforce policies at the gateway level and HTTP Route Level -- before requests ever reach the LLM backend.

{empty} +

=== Gateway API Architecture

_Connectivity Link_ operates at the _Gateway API_ layer -- a _Kubernetes_ standard for managing ingress traffic. It uses a layered model:

[source]
----
GatewayClass          <-- Defines the controller (e.g., OpenShift Gateway Controller)
    |
    v
Gateway               <-- The actual listener (port, protocol, namespace scope)
    |
    v
HTTPRoute             <-- Routing rules (path matching, header modification, backend targets)
----

_Connectivity Link_ attaches its policies (AuthPolicy, TokenRateLimitPolicy) directly to these _Gateway API_ resources. This means security and governance are enforced **at the infrastructure level**, without modifying application code or the AI agent itself.
