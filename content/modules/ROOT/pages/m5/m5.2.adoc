== Test Authentication via the Chat Interface

Now test that the AuthPolicy is working using the same chat interface from Phase 3.

=== Configuring the API Key

Let's configure the chat application to use the API key for authentication. The chat interface is set up to read the API key from the `forage-agent-factory.properties` file. 

. Run the following command from the dev spaces terminal to setup the files needed for this exercise:
+
[.copypaste]
****
step 22
****

. Navigate to the chat folder
+
[.copypaste]
****
cd chat
****




{blank}

. Open the `forage-agent-factory.properties` file under workshop/lab/chat.
+
image::m5/properties_file.png[Properties file location - placeholder,width=80%]
{empty} +

. Locate the `forage.myagent.agent.base.url` property and replace the placeholder *LLM_BASE_URL* with the actual base URL of your HTTP Route attached to the gateway.
+
[.copypaste]
****
http://internal-ai-gateway-openshift-default.ai-gateway.svc.cluster.local/v1
****
+

. Locate the `forage.myagent.agent.api.key` property and replace the placeholder *LLM_API_KEY* with the actual API key we created earlier using the auth policy.
+
[.copypaste]
****
iamafree{user-username}
****
{empty} +
+
Example:
[.copypaste]
****
forage.myagent.agent.api.key=iamafree{user-username}
****

. Make sure the run script is executable
+
[.copypaste]
****
chmod +x ./run.sh
****

. Run the chat application
+
[.copypaste]
****
./run.sh
****

. Open the application in a new browser tab when prompted
+
image::m5/chat_app_launch.png[Chat application launch - placeholder,width=80%]

. You should see the chat interface as shown below. Try sending a message to the LLM for example: `What is Kubernetes?`` and see if you get a response.
+
image::m5/chat_image.png[Chat interface response - placeholder,width=80%]






{empty} +

=== Test Scenarios

Test the following scenarios by updating the API key value and sending a message through the chat interface:

{blank}
. Stop the application by pressing `CTRL+C` in the terminal

. **Invalid credentials** -- Set an invalid API key in `forage-agent-factory.properties`:
+
[.copypaste]
****
invalidkey
****

. Run the application again 
+
[.copypaste]
****
./run.sh
****

Send a message. The request should be **denied** (401 Unauthorized).
+
{empty} +

. Stop the application again by pressing `CTRL+C` in the terminal

. **Valid free-tier key** -- Set the free-tier API key in `forage-agent-factory.properties`:
+
[.copypaste]

iamafree{user-username}
****

. Run the application again 
+
[.copypaste]
****
./run.sh
****

. Send a message. The request should be **allowed** -- you receive a response again from the LLM.
