== Test Authentication via the Chat Interface

{banner-under-construction}

Now test that the AuthPolicy is working using the same chat interface from Phase 3.

=== Configuring the API Key

Let's configure the chat application to use the API key for authentication. The chat interface is set up to read the API key from the `forage-agent-factory.properties` file. 

// . Run the following command from the dev spaces terminal to setup the files needed for this exercise:
// +
// [.copypaste]
// ****
// step 24
// ****

. Navigate to the chat folder
+
[.copypaste]
****
cd chat
****
+
{empty} +

. Open the `forage-agent-factory.properties` file under workshop/lab/chat.
+
image::m5/properties_file.png[Properties file location - placeholder,width=80%]
+
{empty} +

. Locate the `forage.myagent.agent.base.url` property. This is the URL that the chat application will use to connect to the LLM. Since we are running the chat application from within the cluster, we will use the internal service address for the HTTPRoute. Replace the placeholder *REPLACE_WITH_HTTP_BASE_URL* with the following URL:
+
[.copypaste]
****
http://internal-ai-gateway-openshift-default.ai-gateway.svc.cluster.local/{user-username}/v1
****
+
{empty} +

. Locate the `forage.myagent.agent.api.key` property and replace the placeholder *REPLACE_WITH_API_KEY* with the actual API key we created earlier using the auth policy.
+
[.copypaste]
****
iamafree{user-username}
****
+
NOTE: The model name has already been pre-populated in the `forage-agent-factory.properties` file as `qwen3-14b`. This is the model that will be used for the chat interactions in this exercise.
+
{empty} +

. Run the chat application
+
[.copypaste]
****
camel run *
****
+
{empty} +

. Open the application in a new browser tab when prompted
+
image::m5/chat_app_launch.png[Chat application launch - placeholder,width=80%]
+
{empty} +

. You should see the chat interface as shown below. Send the below message  and see if you get a response.
+
[.copypaste]
****
Explain Kubernetes in 20 words
****
+
image::m5/chat_image.png[Chat interface response - placeholder,width=80%]

=== Negative Test Scenarios

Since we have already tested the authentication  with the correct API key, let's test the negative scenarios to confirm that invalid keys are rejected.


. Stop the application by pressing `CTRL+C` in the terminal
+
{empty} +

. Locate the `forage.myagent.agent.api.key` in forage-agent-factory.properties property and set the value to an invalid API key:
+
[.copypaste]
****
invalidkey
****
+
{empty} +

. Run the application again 
+
[.copypaste]
****
camel run *
****
+
{empty} +

. Send the same message. The request should be **denied** with an error message.
+
[.copypaste]
****
Explain Kubernetes in 20 words
****
+
image::m5/invalid_key_response.png[Response with invalid API key - placeholder,width=80%]
+
{empty} +   


. Stop the application again by pressing `CTRL+C` in the terminal
+
{empty} +

. Reset the free-tier API key in `forage-agent-factory.properties` as we will use the valid key for the next section on rate limiting.:
+
[.copypaste]
****
iamafree{user-username}
****
+
{empty} +

. Run the application again and open the chat interface again if needed to prepare for the next section.
+
[.copypaste]
****
camel run *
****
+
{empty} +
