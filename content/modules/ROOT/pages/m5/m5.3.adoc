== Test Authentication via the Chat Interface



Now test that the AuthPolicy is working using the same Rocket.Chat interface from the previous section.

=== Configuring the API Key

Let's configure the chat application to use the API key for authentication. The chat interface is set up to read the API key from the `application.properties` file.

=== Test Authentication with a Valid API Key

Since we have already tested the negative scenario, let's test the positive scenario to confirm that valid keys are accepted.


. Stop the agent by pressing `CTRL+C` in the terminal
+
{empty} +

. Set the valid API key for the `forage.scribe.agent.api.key` property by running the following command in the terminal:
+
[.copypaste]
****
sed -i "s|^forage.scribe.agent.api.key=.*|forage.scribe.agent.api.key=$(oc get secret lab-config -o jsonpath='{.data.config}' | base64 -d | sed -n 's/kuadrant_lite_api_key=//p')|" application.properties
****
+
{empty} +

. You can confirm that the API key value has been updated in the `application.properties` by checking the `forage.scribe.agent.api.key` property. 
+
{empty} +

. Run the application again 
+
[.copypaste]
****
camel run * --local-kamelet-dir ../support/kamelets
****
+
{empty} +

. Go back to the Rocket.Chat interface and send the same message. Rocket.Chat should return a summary generated by the LLM, confirming that the request was authenticated successfully and processed by the agent and the model.
+
[.copypaste]
****
Summarize this customer conversation: Customer: Hi, I received invoice 42 but the quantity for item 3 is wrong. It shows 10 units but I only ordered 5. Agent: I apologize for the error. Let me fix that for you right away. Customer: Also the shipping address is still showing our old office. Agent: I'll update both the quantity and the address. Customer: Thanks, please send me the updated PDF when done.
****   





At this point, authentication and authorization are enforced. Only users with valid API keys belonging to the `lite` or `pro` groups can access the LLM.
