Both Gold and Silver profiles are generated by the same LLM. The AuthPolicy validates the API key presented in each request to determine which tier of service to provide.

This is reflected in the `application.properties` file -- two agent definitions pointing to the same endpoint, each with its own API key:

[#config-snippet]
[source,properties, subs="attributes"]
----
# Gold agent - model configuration
forage.profile-gold.agent.model.kind=openai
forage.profile-gold.agent.model.name=qwen3-14b
forage.profile-gold.agent.base.url=http://internal-ai-gateway-openshift-default.ai-gateway.svc.cluster.local/{username}/v1
forage.profile-gold.agent.api.key=EMPTY

# Silver agent - model configuration
forage.profile-silver.agent.model.kind=openai
forage.profile-silver.agent.model.name=qwen3-14b
forage.profile-silver.agent.base.url=http://internal-ai-gateway-openshift-default.ai-gateway.svc.cluster.local/{username}/v1
forage.profile-silver.agent.api.key=EMPTY
