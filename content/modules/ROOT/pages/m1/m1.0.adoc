== Lab 1 - Ingest Service

{banner-under-construction}

Ingest data from customers, store it in a backend database, and generate PDF invoices — all orchestrated by Apache Camel with secure cross-cluster connectivity powered by {rhsi}.

{empty} +

*The Business Need*

Customers send Purchase Orders (POs) in EDI format to {corp}. These orders need to be ingested, transformed, stored in a backend database, and rendered as PDF invoices for archival. The database resides on a separate infrastructure and is not directly accessible from OpenShift — a secure connectivity solution is required.

{empty} +

*The Technical Solution*

An Apache Camel integration exposes an API endpoint that receives incoming EDI Purchase Orders. The flow transforms the EDI data into XML format, stores it in a backend Database via {rhsi}, and renders PDF invoices that are uploaded to S3 object storage (OpenShift Data Foundation).

{rhsi} enables secure connectivity between the OpenShift cluster and the remote backend database without exposing the database to the public network.

{empty} +
