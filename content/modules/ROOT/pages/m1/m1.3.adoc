== Flow Implementation

Now that you understand the full ingestion flow, let's build it step by step. Here's the Camel route you'll be working with:

{blank}

[.mermaid]
----
flowchart LR
    A((platform-http)) --> B(transform-x12) --> C2 --> D2

    subgraph C2 ["To do: Enable"]
        C[(sql)]
    end

    subgraph D2 ["To do: Enable"]
        D(store-pdf-s3)
    end

    style C fill:#eee,stroke:#ccc,color:#aaa
    style D fill:#eee,stroke:#ccc,color:#aaa
    style C2 fill:#dde4f6,stroke:#6690e6,color:#335
    style D2 fill:#dde4f6,stroke:#6690e6,color:#335
----

Notice that `sql` and `store-pdf-s3` are grayed out — they're disabled in the route. Camel will skip them when running. The idea is to start small: get the first steps working, verify the results, then enable the next ones as you progress.

.What does this command do?
[%collapsible]

* `localhost:8080/ingest/x12` — the Camel route's HTTP endpoint
* `xmllint --xpath '//Seller'` — extracts only the `<Seller>` element from the full XML using an XPath query
* `bat -pP -lxml` — displays the output with XML syntax highlighting
====

* `--properties application.properties` — loads database connection settings
* `--reply` — waits for and displays the response
====

=== Enable the S3 storage step

The last step in the route generates a PDF invoice from the XML and uploads it to S3 object storage.

{blank}

[.mermaid]
----
flowchart LR

In Kaoto, find the `store-pdf-s3` step and enable it. Save the file — Camel will reload the route automatically.

Clicking the `store-pdf-s3` step reveals what happens inside:

{blank}

[.mermaid]
----
flowchart LR
    A(store-pdf-s3) --> B(Template PDF) --> C(generate PDF) --> D(store PDF) --> E(get Link)
----

The sub-route applies an XSLT template to produce a PDF from the XML invoice, uploads it to S3 object storage, and returns a presigned link to the document.

Send another Purchase Order to trigger the complete flow — the response will include a presigned link to the generated PDF:

{blank}

[.copypaste]
****
curl -s \
-H "content-type: text" \
-d @ingest/edi \
localhost:8080/ingest/x12; \
echo
****

--
.What does this command do?
[%collapsible]
====
* `curl -s` — sends the request in silent mode (no progress bar)
* `-H "content-type: text"` — tells the server the payload is plain text (EDI format)
* `-d @edi` — sends the contents of the `edi` file as the request body
* `localhost:8080/ingest/x12` — the Camel route's HTTP endpoint
* `echo` — adds a newline after the response URL for readability
====
