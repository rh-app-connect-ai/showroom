== Kafka to Rocket.Chat

Bridging _Matrix_ and _Rocket.Chat_ bidirectionally requires four flows (as covered in the introduction). The path that carries customer messages to the support agent alone needs two:

* *Flow 1* — Matrix to Kafka _(pre-configured)_
* *Flow 2* — Kafka to Rocket.Chat ✏️ _(you'll build this one)_

NOTE: All four flows follow the same pattern — once you've built one, the others are nearly identical. To keep the lab short, you'll only build flow 2 by hand; the remaining three will be made available and enabled at the end.

*What is a loopback event?*

All systems publish to and listen from a single _Kafka_ topic. This means a message sent from _Rocket.Chat_ ends up in the same topic this flow is consuming — and could circle right back:

[.mermaid]
----
flowchart LR
    RC -->|"message"| K -->|"❌ same event"| RC

    RC@{ img: "../_images/m4/rocketchat-icon.svg", label: "Rocket.Chat", pos: "b", h: 100, w: 100, constraint: "on" }
    K@{ img: "../_images/common/kafka-icon.svg", label: "Kafka topic", pos: "b", h: 100, w: 80, constraint: "on" }

    style RC fill:none,stroke:none
    style K fill:none,stroke:none

    linkStyle 0 stroke:#66bb6a,stroke-width:2px
    linkStyle 1 stroke:#d32f2f,stroke-width:2px,stroke-dasharray:5
----

The filter you'll define prevents this by checking the `source` field in each event and blocking those that originate from _Rocket.Chat_.

=== Define the filter

Events arriving from _Kafka_ can originate from any source system. Without a filter, a message originally sent from _Rocket.Chat_ would flow through _Kafka_ and be pushed back to _Rocket.Chat_ — creating a loop:

  * _Rocket.Chat -> Kafka -> Rocket.Chat_ ⟵ loopback!

{blank}

The filter prevents this by checking where the event came from.

1. Click the bar pass:[<code style="background-color: #B0380C; color: white;">&nbsp;To do: Filter&nbsp;</code>]
+
Ensure the `Required` tab is selected. You should see the `JQ` expression language selected by default.
+
{blank}
+
1. Copy and paste the following expression:
+
{blank}
+
[.copypaste]
****
pass:[.source | test("rocketchat")]
****
+
NOTE: The expression picks the `source` field from the _Kafka_ event and checks if it originates from _Rocket.Chat_. Events from _Rocket.Chat_ are filtered out to prevent loopbacks.
+

Once ready, send a test message to _Kafka_:

{blank}

[.copypaste]
****
+++camel cmd send k2r --body \
'{
  "user":   "test",
  "source": "test",
  "text":   "test message"
}'+++
****

[TIP]
--
.What does this command do?
[%collapsible]
====
* `camel cmd send` — sends a message to a running Camel process
* `k2r` — the name of the target process
* `--body '{...}'` — a JSON payload simulating a Kafka event. Only `user`, `source`, and `text` are needed — the target channel is hardcoded in the mapping
====
