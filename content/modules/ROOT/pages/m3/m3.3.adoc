== Deploy to OpenShift

Now that the _Kafka_ to _Rocket.Chat_ flow has been validated locally, it's time to deploy it on _OpenShift_.

{empty} +

=== Stop the local process

Make sure you have stopped your local _Camel_ instance. If it is still running, press kbd:[Ctrl+C] to stop it.

{empty} +

=== Deploy the flow

Use the command below to deploy on _OpenShift_:

{blank}

[.copypaste]
****
labdeploy k2r
****

{blank}

NOTE: Be patient, this action may take some time to complete.

{empty} +

=== Check the deployment

Monitor the state of the resource with the following command:

{blank}

[.copypaste]
****
oc get deployment/k2r
****

{blank}

When the deployment is ready, the command outputs something similar to:

++++
<pre style="background-color: #272822; padding: 10px 15px; line-height: 1.4; white-space: pre-wrap; font-family: monospace; font-size: 14px; color: white">NAME   READY   UP-TO-DATE   AVAILABLE   AGE
k2r    1/1     1            1           31s</pre>
++++

{empty} +

=== Test the deployed flow

The _Kafka_ cluster lives in your namespace, so the deployed pod consumes from the same topic you used locally. Send a test message:

{blank}

[.copypaste]
****
camel cmd send k2r --body '{"user":"test","channel":"room2","source": "test","text": "Hello from OpenShift"}'
****

{empty} +

Open your _Rocket.Chat_ room — you should see the message delivered by the deployed flow:

{empty} +

[.success]
The flow is running on _OpenShift_. In the next section, you'll wire up the other half — _Matrix_ to _Kafka_.

[.fail]
Check the pod logs with `oc logs deployment/k2r` for errors.
